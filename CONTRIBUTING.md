## Contributions

Hello!

Welcome to our project! We accept all kinds of feedback here. Before requesting or submitting anything, please make sure your comments abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

### Arms-Length Contributors

#### Pull Requests
If you would like to contribute to our project, you will first need to fork the GitHub repository. This can be done by clicking the "Fork" button at the top right of the page. This will create a copy of the repository under your own GitHub account, which you can then clone:

``` bash
git clone https://github.com/YOUR_USERNAME/dsci-310-group-15
```

Then, ensure that you have installed all the necessary dependencies. Instructions are available in [README.md](README.md)

Then, make a new branch:

``` bash
git switch -c your-new-branch
```

When you are ready, commit your branch with: 
```bash
git add your-file 
git commit -m "Your message about your changes"
git push origin your-branch-name
```

Then, you will need to submit a [pull request](https://github.com/UBC-DSCI-310-2025W2/dsci-310-group-15/pulls) on GitHub, which will request that your changes be merged into the main branch. To do this:
1. Navigate to the main page of the repository
2. Click on the Branch menu, choose the branch with your changes
3. Click "Compare & pull request" on the yellow banner at the top of the page.

Please include an informative description in your pull request detailing what changes were made. Also, please open an [issue](https://github.com/UBC-DSCI-310-2025W2/dsci-310-group-15/issues) so we can be notified of your pull request. 

 We try to get back to pull requests ASAP (ideally within 2-4 weeks), but please be patient if it looks like we haven't done anything yet. If you haven't seen any updates after 4 weeks, you can submit a new issue/pull request.

#### Issues and Requests
If you notice any bugs or issues in our code, or would like to request an additional feature/script/any outputs, feel free to open up an [issue](https://github.com/UBC-DSCI-310-2025W2/dsci-310-group-15/issues).


### Core Members
Please follow the GitHub flow workflow. Create a new branch and open a PR for changes. Other members will review the changes and make requests as needed. Your PR will be merged if it is approved.
